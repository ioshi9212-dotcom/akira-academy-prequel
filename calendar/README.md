# Calendar Module

Календарь — это источник важных крючков дня, а не готовый сценарий.

## Как читать календарь

1. Сначала читать `state/current_state.json`.
2. Взять `current_date`.
3. Открыть `calendar/calendar_index.yaml`.
4. Найти текущую дату.
5. Открыть файл дня из `day_file`.
6. Сверить `state/calendar_runtime.json`, если он есть.
7. Только потом писать сцену.

## Главные правила

- Календарь задаёт важные крючки дня.
- Календарь не запрещает другие события, если они логичны и интересны.
- Календарь не заменяет живой сюжет.
- Главная цель сцены: интересный интерактивный сюжет, диалоги, выбор игрока, отношения, конфликт, репутация и последствия.
- Все персонажи ведут себя строго по своим карточкам.
- Если персонаж указан в `first_introduction_ids`, это его первое обязательное сюжетное появление.
- После первого появления персонаж может появляться в следующих днях логично и интересно без повторного указания.
- Отсутствие персонажа в файле дня не означает запрет на его появление.
- До первого появления персонаж не должен появляться активно без прямого разрешения игрока, state или календаря.
- Не тянуть день по минутам.
- Если день перегружен событиями, мягко вести его к завершению: усталость, вечер, мысль Акиры, переход ко сну или следующий значимый момент.
- Если в календаре мало крючков, ИИ может развить день, если может сделать это интересно.
- Если интересного развития нет, лучше коротко завершить блок или день.
