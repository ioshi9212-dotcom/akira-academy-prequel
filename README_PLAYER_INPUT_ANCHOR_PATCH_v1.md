# academy_prequel_player_input_anchor_patch_v1

Назначение: исправить проблему, когда сцена то сжимается в пересказ хода игрока, то разрастается в длинный NPC-диалог, где Акира теряет право ответить.

## Что меняется

Патч добавляет жёсткое правило Player Input Anchor:

- всё вне скобок в сообщении игрока = точная реплика Акиры;
- всё в скобках = действие, жест, пауза, состояние, намерение или движение Акиры;
- паузы между репликами Акиры заполняются реакцией мира/NPC;
- Ливия и Кир не отвечают за Акиру на прямой вызов;
- если NPC бросает Акире прямой вопрос, вызов, социальный укол или провокацию, сцена останавливается на выборе игрока;
- нельзя делать 10+ реплик NPC подряд после сильной реплики Акиры без нового выбора игрока.

## Файлы

- `gpt/locks/player_input_anchor_lock.md` — новый lock с основной логикой.
- `gpt/locks/gameplay_response_gate.md` — усилен gate: player input anchor, stop point, agency check.
- `gpt/scene_format.md` — добавлен подробный раздел, как читать скобки/реплики и где останавливать сцену.
- `app/prompt_builder.py` — prompt_preview теперь явно требует chunked loading и player input anchor protocol.

## Что НЕ меняется

- Actions schema не меняется.
- Railway config не меняется.
- API endpoints не меняются.
- Персонажи/канон не меняются.
- State не меняется.

## Тестовый сценарий

Игровой ввод:

```txt
Баскетбол честнее людей. Мяч хотя бы не врёт, когда падает. (широко довольно улыбнуться, потом быстро убрать улыбку и посмотреть на Хару без интереса) Зрители платят тишиной. У тебя с этим сложно. (Идти к входу B-2 с Ливией и Киром, оставив вызов на после проверки.)
```

Ожидаемое поведение:

- обе реплики вставлены как реплики Акиры;
- скобки превращены в паузы/действия;
- мир реагирует между репликами;
- сцена не пересказывает ход кратко;
- сцена не уходит в 20 реплик NPC без Акиры;
- если Киара/Хару/богатый новичок бросают прямой вызов Акире, сцена останавливается на выборе ответа.
