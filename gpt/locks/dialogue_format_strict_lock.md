# Lock: строгий формат реплик

Этот файл имеет высокий приоритет для любого сценового ответа.

## Главный принцип

Шапка сцены недостаточна.

Если в сцене есть реплики, каждая реплика должна быть оформлена строго так:

```txt
**Имя или видимый дескриптор** — Реплика. (*короткая ремарка: тон, взгляд, пауза, жест*)
```

Жирное имя обязательно.

Длинное тире обязательно.

Ремарка необязательна, но если она есть — только короткая и в скобках курсивом.

## Правильно

```txt
**Ливия** — Ты опять смотришь так, будто уже нашла три выхода. (*усмехается*)

**Хару** — Это талант или угроза? (*смотрит с интересом*)

**Акира** — Привычка.
```

Если имя персонажа Акира ещё не знает, использовать видимый дескриптор:

```txt
**Рыжий старшекурсник** — Новенькие? (*останавливается у края площадки*)
```

## Неправильно

```txt
Ливия — Ты опять смотришь так...

Ливия: Ты опять смотришь так...

- Ты опять смотришь так, — сказала Ливия.

Ливия усмехнулась: Ты опять смотришь так...
```

## Описания отдельно

Действия и атмосфера не должны смешиваться с репликами.

Правильно:

```txt
*Ливия опирается плечом о стену и быстро смотрит на Акиру.*

**Ливия** — Не делай вид, что ты этого не заметила.
```

Неправильно:

```txt
Ливия опирается о стену и говорит — Не делай вид...
```

## Проверка перед отправкой

Перед отправкой сцены ИИ обязан проверить:

- каждая реплика начинается с `**`;
- имя/дескриптор закрыт `**`;
- после имени стоит длинное тире `—`;
- реплика не оформлена обычным текстом;
- нет формата `Имя:`;
- нет формата `Имя —` без жирного имени;
- нет книжного формата с кавычками вместо сценового диалога.

Если хотя бы одна реплика оформлена не так — сцену нужно переписать до отправки.

Ход с неправильным форматом реплик считается технически неправильным, даже если шапка оформлена верно.
