# Lock: academy start cleanup 1198-08-15

Этот lock фиксирует чистку стартовой сцены Академии Астрейн после расхождения файлов.

## Canonical start location

15 августа 1198 стартовая сцена начинается не в главном дворе.

Правильная точка:

```txt
academy_back_court_exit
задний выход Академии со стороны корта / баскетбольные площадки
```

Причина маршрута:

- Акира и Ливия входят через менее людную сторону;
- маршрут выбран по совету Рэя;
- рядом баскетбольные площадки / корт;
- первый социальный крючок может идти через Хару и баскетбол.

## Akira start outfit

На старте 15 августа Акира НЕ в форме Академии.

Правильная одежда:

```txt
кроссовки, чёрные штаны карго, топ, бордовый худи
```

Писать форму Академии как надетую можно только если:

- игрок явно сказал, что Акира переоделась;
- current_state содержит `uniform_worn: true`;
- сцена явно применяет переодевание и state сохраняет это изменение.

## Received uniform is not worn uniform

Если Академия выдала форму, хранить отдельно:

```json
{
  "current_outfit": "кроссовки, чёрные штаны карго, топ, бордовый худи",
  "received_uniform": ["..."],
  "uniform_worn": false
}
```

Полученная форма может быть:

- в пакете;
- в руках;
- в багаже;
- на кровати в комнате;
- рядом как предмет.

Она не появляется на теле без действия игрока.

## Phone / visible inventory

Телефон Акиры на старте внутри багажа.

Не писать телефон в шапке как предмет при себе, пока Акира не достала его действием.

Правильно на старте:

```txt
◈ багаж с личными вещами
```

Неправильно на старте:

```txt
◈ телефон · ключ-карта · кофе
```

Если ключ-карта или кофе появятся позже, их нужно получить сценой или state.

## Calendar conflict rule

Активный календарь первого дня:

- `canon/august_15_calendar.yaml`
- `state/academy_schedule.json`, блок `1198-08-15`

Устаревший patch-файл:

```txt
state/academy_schedule_patch_v2.json
```

не использовать как источник active plan.

## Location registry rule

`canon/academy_locations.md` — базовая карта.

`state/location_registry.md` — сыгранные/уточнённые локации.

Не дублировать весь список базовых локаций в registry.
