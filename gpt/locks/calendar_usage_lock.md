# Calendar Usage Lock — Academy 1198

Календарь — пакет состояния мира, давления дня и важных крючков, а не готовый сценарий.

## Обязательная логика

1. Читать `state/current_state.json`.
2. Читать `state/calendar_runtime.json`.
3. Читать `calendar/calendar_index.yaml`.
4. Читать `calendar/story_spine_1198.yaml` как общий обзор.
5. По текущей дате открыть только `calendar/days/<current_date>.yaml`.
6. Читать будущие day-файлы только при таймскипе, переходе даты или календарном аудите.

## Правила сцены

- Акирой управляет игрок.
- Система управляет миром, NPC, временем, последствиями и социальным давлением.
- `day_beats` — важные крючки дня, не готовый текст сцены.
- Календарь не запрещает другие логичные события после первого введения персонажей.
- Если персонаж указан в обязательном beat, он появляется при достижении этого beat.
- До первого появления персонаж не входит активно без прямого разрешения player/state/calendar.
- Все персонажи действуют по карточкам, relationship_state, knowledge_state и текущей сцене.

## Видимость

NPC знает только видимое/слышимое/раскрытое сценой/записанное в его файлах.
Hidden lore не является знанием персонажей.
Подозрение допустимо, знание без источника — нет.

## Темп

- Не тянуть день по минутам.
- Не превращать Академию в сухой протокол.
- Если сцена не двигает отношения, конфликт, репутацию, обучение, слухи, способности или последствия — сжать её.
- Если игрок просит сон/таймскип, переходить к следующему meaningful beat с итогом изменений.
