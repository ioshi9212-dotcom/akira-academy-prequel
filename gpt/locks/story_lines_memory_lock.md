# Lock: память по сюжетным линиям, без одноразовых state-файлов

Этот файл имеет высокий приоритет для записи памяти после сцены.

## Главная цель

Не создавать отдельный state-файл под каждое событие сцены.

Неправильно:

```txt
state/breakfast_kiara_marked_by_voice_1198_08_17.json
state/lunch_haru_coffee_debt_1198_08_17.json
state/corridor_npc_conflict_1198_08_17.json
```

Так память распадается на куски, которые потом трудно подтягивать.

Правильно:

```txt
state/story_lines.json
```

В нём событие записывается в нужную линию по ID:

- `line_haru`
- `line_raiden`
- `line_livia`
- `line_kiara`
- `line_academy`
- `line_reputation`
- `line_social_media_rumors`
- `line_obligations`

## Разделение анкеты и событий

Карточка персонажа хранит личность, характер, роль, внешность, энергию и базовые связи.

Пример:

```txt
characters/main/kiara_volt.md
```

`state/story_lines.json` хранит только события и последствия:

- что произошло;
- кто видел;
- кто что узнал;
- какие отношения изменились;
- какая нитка открыта;
- какой долг/обещание получил прогресс;
- какой слух или репутационный крючок появился.

Не копировать характер и анкету персонажа в `story_lines.json`.

## Даты событий и календарь

Каждое событие в `story_lines.shared_events`, `daily_timeline`, `events`, `open_threads`, `obligations` и `progress_log` обязано иметь абсолютную дату.

Формат внутри данных:

```txt
1198-08-15
```

В коротком summary можно писать человекочитаемо:

```txt
15 августа 1198 — Акира и Хару сыграли в баскетбол; Хару проиграл ставку на кофе.
```

Перед любой репликой персонажа со словами:

- сегодня;
- вчера;
- позавчера;
- несколько дней назад;
- неделю назад;
- тогда утром;
- на днях;

ИИ обязан сравнить `event.date` из `state/story_lines.json` с `state/current_state.json.date`.

Нельзя опираться на ощущение времени из памяти чата.

Правило относительных дат:

- та же дата — `сегодня`, `утром`, `днём`, `вечером`;
- 1 день назад — `вчера`;
- 2 дня назад — `позавчера` или `два дня назад`;
- 3–6 дней назад — `несколько дней назад` или точное число дней;
- 7 дней назад — `неделю назад`;
- больше 7 дней — точная дата или период.

Запрещено говорить `вчера`, если между `state/current_state.json.date` и `event.date` прошло больше одного дня.

Пример:

Если текущая дата `1198-08-20`, а событие записано как `1198-08-15`, персонаж должен сказать:

```txt
пять дней назад
```

или:

```txt
ещё пятнадцатого
```

но не:

```txt
вчера
```

## Daily timeline

`state/story_lines.json` содержит `daily_timeline`.

Туда нужно добавлять короткую запись по дате:

```json
{
  "1198-08-15": [
    {
      "event_id": "basketball_haru_bet_1198_08_15",
      "summary": "Акира и Хару сыграли в баскетбол; Хару проиграл ставку и должен неделю приносить кофе без сахара.",
      "line_refs": ["line_haru", "line_obligations"]
    }
  ],
  "1198-08-17": [
    {
      "event_id": "breakfast_kiara_voice_1198_08_17",
      "summary": "Утром высокая светловолосая курсантка с дорогими украшениями спровоцировала Акиру за завтраком; имя Акира ещё не знает.",
      "line_refs": ["line_kiara", "line_livia", "line_reputation"]
    }
  ]
}
```

`daily_timeline` нужен, чтобы быстро понять, что было в конкретный день, и не путать относительные даты.

## Shared event

Если несколько персонажей видели одно и то же событие, не нужно дублировать полный пересказ в каждую линию.

Нужно создать одну короткую запись в `shared_events`:

```json
{
  "event_id": "breakfast_kiara_voice_1198_08_17",
  "date": "1198-08-17",
  "time": "утро / завтрак",
  "location": "столовая",
  "participants": ["akira", "livia_cross", "kiara_volt"],
  "witnesses": ["livia_cross"],
  "summary": "Высокая светловолосая курсантка с дорогими украшениями спровоцировала Акиру за завтраком. Акира не ответила вслух, но запомнила голос и манеру давления. Имя Киары Акира ещё не знает.",
  "known_name_to_akira": false,
  "public_name": "kiara_volt",
  "stable_descriptor_for_akira": "высокая светловолосая курсантка с дорогими украшениями"
}
```

А в линиях добавить короткие ссылки:

```json
{
  "event_ref": "breakfast_kiara_voice_1198_08_17",
  "effect": "Акира отметила голос и манеру давления; Ливия раздражена провокацией."
}
```

## Пример: Киара утром

Утренняя перепалка с девушкой с дорогими украшениями должна записываться не отдельным файлом, а так:

- в `daily_timeline["1198-08-17"]` — короткое событие дня;
- в `shared_events` — короткое событие;
- в `line_kiara.events` — что Киара вошла в поле Акиры как безымянная раздражающая курсантка;
- в `line_livia.events` — что Ливия видела провокацию и раздражена;
- в `line_reputation` или `line_social_media_rumors` — только если сцена могла стать слухом;
- в `relationships.json` — кратко создать/обновить пару, если уже есть значимая динамика;
- в `knowledge_state.json` — что Акира и Ливия пока не знают имени, но Акира узнаёт голос/манеру.

## Пример: долг Хару за кофе

Ставку Хару не нужно разбрасывать на отдельные файлы по каждому кофе.

Правильно хранить одну открытую obligation в `line_obligations.open_obligations`:

```json
{
  "obligation_id": "haru_coffee_debt_1198_08_15",
  "from": "haru_foster",
  "to": "akira",
  "created_at_date": "1198-08-15",
  "summary": "Хару проиграл ставку после баскетбола и должен неделю приносить Акире кофе без сахара.",
  "total_required": 7,
  "completed": 1,
  "remaining": 6,
  "progress_log": [
    {
      "date": "1198-08-16",
      "event": "первый кофе принесён и принят"
    }
  ],
  "status": "open"
}
```

Когда Хару приносит второй кофе — обновить эту же запись, а не создавать новый файл.

## Каждые 15 ходов — сверка и сжатие

`story_lines.turn_counter.since_last_compaction` увеличивается после каждого игрового хода.

Когда значение достигает 15:

1. прочитать последние события в `story_lines.json`;
2. закрыть завершённые нитки;
3. обновить progress obligations;
4. перенести повторяющиеся мелкие события в `compacted_summary`;
5. убрать лишние мелкие дубли из `events`, оставив только актуальные;
6. сбросить `since_last_compaction` в 0;
7. записать краткую заметку в `last_compaction_note`.

Пример с долгом Хару:

Было:

- Хару должен 7 кофе;
- кофе 1 отдан;
- кофе 2 отдан;
- кофе 3 отдан;
- кофе 4 отдан.

После сжатия:

```txt
Хару проиграл ставку и уже отдал 4 из 7 кофе. Осталось 3. Долг открыт.
```

Не нужно тянуть все четыре отдельные сцены, если важен только прогресс долга.

## Что запрещено

Запрещено:

- создавать новый `state/*.json` под каждую сцену;
- хранить событие только в тексте ответа;
- копировать характер персонажа в story_lines;
- дублировать один и тот же event полностью в нескольких местах;
- создавать отдельный файл на каждый принесённый кофе;
- терять, кто видел событие и кто знает имя;
- называть персонажа по имени в сцене, если Акира его ещё не знает;
- заменять карточку персонажа событием;
- записывать event без абсолютной даты;
- использовать относительные даты без сравнения `event.date` из `state/story_lines.json` с `state/current_state.json.date`.

## Короткая формула

Анкета персонажа — в `characters/main/<id>.md`.

События и последствия — в `state/story_lines.json`.

Один event — один раз в `shared_events`, дальше только ссылки и короткие эффекты по линиям.

Каждый event имеет дату. Относительные слова вроде `вчера` считаются через сравнение `event.date` и `state/current_state.json.date`, а не из памяти чата.
